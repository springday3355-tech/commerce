# 실시간 커머스 플랫폼

## 필수 기능
1. Product라는 설계도로 노트북,스마트폰,맥북,에어팟의 실제 객체를 데이터화 함
2. ArrayList라는 유연한 주머니에 여러 상품들을 목록에 담음
3. 향상된 for문으로 번호를 일일이 세지 않고 예쁜 가격표와 상품 목록을 출력함
4. DecimalFormat을 사용해 사용자가 숫자를 보기 편하게 함
5. 사용자 상세조회의 모든 기능을 Main(사장)에 넣지 않고 CommerceSystem(점장)클래스에 일을 대신 맡김(객체지향 설계)
6. new 키워드는 설계도에서 진짜 물건을 찍어주는 주문서 같은 존재인데 new를 사용해서 메모리에 객체를 실제로 올려야 기능을 사용 할 수 있다
7. 클래스간 데이터를 안전하게 전달해주기 위해 생성자 사용
8. 클래스를 분리하고 구조화 해 Main은 실행만 담당하고, 실질적인 로직은 CommerceSystem으로 옮겨담음
9. 데이터 모델링을 해 Product뿐만 아니라 Category,Customer 클래스를 생성하여 데이터를 객체 단위로 관리하는 법을 익힘
10. List를 CommerceSystem이 직접 갖지 않고 Category 클래스가 관리하도록 구조를 개선함
11. 필드를 Private으로 선언하고 Getter를 통해 안전하게 접근하는 캡슐화를 적용


## 도전기능
Lv.1
1. 사용자 입력 예외처리: 사용자가 목록에 없는 번호를 눌러도 프로그램이 터지지 않게 if문과 continue로 가드침
2. 객체 간 메서드 호출 : CommerceSyetem에서 cart.dispalyCart()와 cart.addProduct()를 호출하여 클래스끼리 대화시킴
3. 장바구니 로직 구현: total 변수를 0으로 초기화하고 for문을 돌며 누적합산(+=)하여 계산하는 로직 구현함
4. 재고관리 : 상품을 담기 전 getStock() > 0인지 체크하여 실제 비즈니스 로직이 어떻게 돌아가는지 체득함
5. else if 를 사용자가 잘못입력했을 때를 대비해 예외처리
